## 分析

### 问题根因

1. **视觉问题**：T5 是灰盒原型，确实只用了方框和圆圈。但用户期望看到"东西归位"的意象（书桌/床铺/衣柜），而不是抽象几何。
2. **玩法单调**：单纯的"移动到目标按空格"缺乏：
   - 节奏感 / 音乐配合
   - 递进变化（难度曲线太平）
   - 正向反馈循环（combo、连击、得分爆发）
   - 失败的情绪价值（现在的失败只是"早了/晚了"，没有戏剧化）

### 迭代方向（在计划范围内）

在"单键时机"机制内做深，不引入新机制：

1. **加 Combo 系统**：连续命中有连击数，combo 越高得分倍率越高，错过打断——制造"不想断连"的张力
2. **加节奏/节拍提示**：用 Web Audio 合成节拍声，让玩家能"听"到时机——把"看"时机变成"听+看"时机
3. **加动态目标**：目标不是静止的，会轻微摆动或收缩——增加挑战性
4. **加视觉意象**：
   - 守序者路径："凌乱的东西"（书本、袜子、杯子）从乱的位置飞回柜子/书桌
   - 闯入者路径："整齐的东西"在柜子上，玩家要小心不碰乱
   - 命中时：东西"啪"地归位 + 小动画
   - 失败时：东西飞走/打碎 + 夸张动画
5. **加难度递进**：不是单纯窗口缩小，而是：
   - 目标开始移动
   - 出现干扰物（假目标）
   - 速度波动（忽快忽慢）

### 与原 spec 的对应

- 这些改进都在原 spec 的 In-scope 内："成功/失败反馈（视觉 + 音效占位）"、"难度递增"、"单键时机核心机制"
- 不引入新机制，只是让现有机制更有表现力
